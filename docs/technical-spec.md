# Техническое задание на учебную практику (БПИ24)

## 1. Цель и задачи

**Цель:** Закрепление теории через разработку полнофункционального поискового веб-приложения.

**Задачи:**
- Микросервисная архитектура.
- Полнотекстовый поиск (Elasticsearch).
- UI для загрузки и поиска.
- Контейнеризация (Docker).
- CI/CD, мониторинг.
- Документирование.

## 2. Команда и роли

| Роль | Код | Обязанности |
|------|-----|-------------|
| Бэкенд | BE | API, Elasticsearch, бизнес-логика, БД |
| Фронтенд | FE | UI, вёрстка, клиент-серверное взаимодействие |
| DevOps | DO | Docker, CI/CD, мониторинг, окружение |
| QA | QA | Автотесты, тестовые данные, качество, документация |

## 3. Функциональные требования

### 3.1. Бэкенд

#### Загрузка документов

| ID | Требование | Приоритет |
|----|------------|-----------|
| BE-01 | `POST /api/v1/documents/upload` | Высокий |
| BE-02 | Валидация: PDF/DOCX, ≤20 МБ, иначе 400 | Высокий |
| BE-03 | Генерация UUID для документа | Высокий |
| BE-04 | Извлечение текста (pdfplumber, python-docx) | Высокий |
| BE-05 | Чанкинг: 1000 символов, overlap 100 | Средний |

#### Индексация и поиск

| ID | Требование | Приоритет |
|----|------------|-----------|
| BE-06 | Elasticsearch с analysis-ru, индекс `documents` | Высокий |
| BE-07 | Индексация чанков с метаданными | Высокий |
| BE-08 | `GET /api/v1/search?q={query}` multi-match | Высокий |
| BE-09 | Ответ JSON: chunk_id, file_name, page, text, score | Высокий |

#### API

- OpenAPI 3.0 (Swagger UI по `/docs`).
- Статусы: 200, 400, 404, 500.

## 4. Технологический стек

- **Backend:** Python, FastAPI, PostgreSQL, Elasticsearch, Redis
- **Frontend:** TypeScript, React (Vue 3), Vite, Nginx
- **DevOps:** Docker, Docker Compose, GitHub Actions, Prometheus, Grafana
- **Testing:** Pytest, Playwright

## 5. Качество кода

- PEP8, докстринги, понятные имена.
- Без бинарников (кроме `tests/fixtures`).
- Коммиты: `feat:`, `fix:`, `docs:`.
