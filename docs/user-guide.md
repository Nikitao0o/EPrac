# Руководство пользователя

## Что сейчас доступно

На текущем этапе можно проверить загрузку документов через API. Поддерживаются файлы PDF и DOCX размером до 20 МБ.

После загрузки система возвращает UUID документа, имя файла и размер. Поиск по документам, индексация, подсветка результатов и полноценный frontend пока находятся в разработке.

## Как запустить

1. Создать файл `.env` на основе `.env.example`.
2. Запустить проект через Docker Compose.
3. Открыть Swagger UI: `http://localhost:8000/docs`.

## Как загрузить документ

1. В Swagger UI найти метод `POST /api/v1/documents/upload`.
2. Нажать `Try it out`.
3. Выбрать PDF- или DOCX-файл.
4. Выполнить запрос.

В успешном ответе должны быть поля:

- `document_id`
- `filename`
- `size_bytes`

## Возможные ошибки

| Ситуация | Результат |
| --- | --- |
| Загружен файл неподдерживаемого формата | `HTTP 400` |
| У файла PDF/DOCX неверный `content-type` | `HTTP 400` |
| Размер файла больше 20 МБ | `HTTP 400` |

## Примечание

Тестовые файлы для ручной проверки лежат в `backend/tests/fixtures`. После реализации frontend и поиска это руководство нужно будет дополнить инструкцией по полному пользовательскому сценарию.
