# Руководство пользователя

## Что сейчас доступно

В проекте доступна загрузка PDF/DOCX-документов, извлечение текста, индексация и поиск по загруженным данным.

Работать с API можно через Swagger UI: `http://localhost:8000/docs`.

## Как запустить

1. Создать файл `.env` на основе `.env.example`.
2. Запустить проект через Docker Compose:

```bash
docker compose up --build
```

3. Открыть Swagger UI: `http://localhost:8000/docs`.

## Как загрузить документ

1. В Swagger UI открыть метод `POST /api/v1/documents/upload`.
2. Нажать `Try it out`.
3. Выбрать PDF- или DOCX-файл размером до 20 МБ.
4. Выполнить запрос.

В успешном ответе должны быть поля:

- `document_id`
- `filename`
- `size_bytes`
- `total_chunks_indexed`

## Как выполнить поиск

1. В Swagger UI открыть метод `GET /api/v1/search`.
2. В параметре `q` указать поисковый запрос.
3. При необходимости заполнить `page`, `page_size` или `file_name`.
4. Выполнить запрос.

В ответе возвращается список `results`. Для каждого результата отображаются:

- `chunk_id`
- `file_name`
- `page`
- `text`
- `score`

## Как посмотреть список документов

Метод `GET /api/v1/documents` возвращает список файлов, которые есть в индексе Elasticsearch.

## Возможные ошибки

| Ситуация | Результат |
| --- | --- |
| Загружен файл неподдерживаемого формата | `HTTP 400` |
| У файла PDF/DOCX неверный `content-type` | `HTTP 400` |
| Размер файла больше 20 МБ | `HTTP 400` |
| Поисковый запрос пустой | `HTTP 400` |
| Elasticsearch недоступен | Возможна ошибка обработки запроса |

## Примечание

Для ручной проверки можно использовать файлы из `backend/tests/fixtures`. Проверочный сценарий: загрузить `valid-lecture.docx`, выполнить поиск по `EPrac`, затем открыть `GET /api/v1/documents` и убедиться, что файл появился в списке.

Если `docker compose up --build` не запускается, сначала нужно убедиться, что Docker Desktop/engine включён. Если порт `8000` занят, можно временно указать другой `BACKEND_HOST_PORT` в локальном `.env`, например `8001`.
