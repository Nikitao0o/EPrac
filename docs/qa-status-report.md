# QA-отчёт

## Что проверялось

На текущем этапе в проекте реально доступна только базовая часть backend: загрузка одного PDF/DOCX-файла через `POST /api/v1/documents/upload`, проверка формата, проверка размера и генерация `document_id`.

Поэтому основной фокус проверки сейчас был на загрузке файлов и подготовке тестовых данных. Полный сценарий с поиском пока проверить нельзя, потому что ещё нет парсинга документов, индексации, frontend-части и эндпоинта поиска.

## Что уже можно считать готовым по QA

| Задача | Результат |
| --- | --- |
| QA-03. Набор тестовых документов | Подготовлены файлы для проверки загрузки: валидные PDF/DOCX, пустые файлы, битые файлы и неподдерживаемый формат. Файлы лежат в `backend/tests/fixtures`. |
| QA-01. Тесты валидации | Добавлены pytest-тесты для текущего upload endpoint. Проверяются успешная загрузка PDF/DOCX, ошибка на неподдерживаемый формат, ошибка на неверный `content-type`, превышение размера и наличие UUID в ответе. |

Полностью закрывать `QA-01` пока рано: в задании там также указаны тесты парсинга, а самих парсеров PDF/DOCX в проекте ещё нет.

## Проверки для upload API

| Проверка | Ожидаемый результат |
| --- | --- |
| Загрузить корректный PDF | `HTTP 200`, в ответе есть `document_id`, `filename`, `size_bytes`. |
| Загрузить корректный DOCX | `HTTP 200`, в ответе есть `document_id`, `filename`, `size_bytes`. |
| Загрузить `.txt` | `HTTP 400`. |
| Загрузить PDF с неверным `content-type` | `HTTP 400`. |
| Загрузить файл больше лимита | `HTTP 400`. |

Эти проверки покрыты автотестами в `backend/tests/test_upload_api.py`.

## Что пока не проверялось

| Задача | Почему пока не проверяется |
| --- | --- |
| QA-02. E2E-сценарий | Нет полного пользовательского сценария: загрузка пока не приводит к парсингу, индексации и поиску. |
| QA-04. Нагрузочное тестирование | Нагрузку нужно давать на поиск, а `/api/v1/search` ещё не реализован. |
| QA-05. Precision@3 | Метрику можно считать только после появления поисковой выдачи и эталонного набора запросов. |
| Финальная версия QA-06 | Руководство пользователя сейчас описывает только доступную часть API. После появления frontend и поиска его нужно будет дополнить. |

## Замечания

- Сейчас upload endpoint проверяет расширение файла и `content-type`, но не проверяет реальное содержимое PDF/DOCX.
- Frontend пока представлен контейнером-заглушкой, поэтому ручные проверки интерфейса не проводились.
- Swagger UI доступен через `/docs`, его можно использовать для ручной проверки загрузки.
- Мониторинг подключён, но отдельные метрики поиска появятся только после реализации поиска.
