# QA-отчёт

## Что проверялось

На текущем этапе в проекте уже есть основной backend flow: загрузка PDF/DOCX, извлечение текста, разбиение на чанки, индексация в Elasticsearch и поиск через `GET /api/v1/search`.

Основной фокус QA сейчас — автотесты backend-части, тестовые документы и проверка того, какие требования уже можно закрывать без полноценного E2E-стенда.

## Что уже можно считать готовым по QA

| Задача | Результат |
| --- | --- |
| QA-01. Юнит-тесты | Добавлены pytest-тесты для upload/search/parser/es_client. Парсер дополнительно проверяется на реальных PDF/DOCX-фикстурах. В CI проверяется порог покрытия 50%. |
| QA-03. Набор тестовых документов | Подготовлены валидные PDF/DOCX, пустые файлы, битые файлы и неподдерживаемый формат. Файлы лежат в `backend/tests/fixtures`. |
| QA-06. Руководство пользователя | Руководство обновлено под доступные API загрузки, поиска и списка документов. |
| QA-04. Нагрузочное тестирование | Добавлен сценарий на 50 одновременных поисковых запросов и отчёт с результатами. |
| QA-05. Precision@3 | Добавлен оценочный сценарий на 10 эталонных запросов. Итоговая метрика: 1.00. |

## Что покрыто автотестами

| Область | Проверки |
| --- | --- |
| Загрузка документов | Успешная загрузка PDF/DOCX, неподдерживаемый формат, неверный `content-type`, превышение лимита размера. |
| Парсер | Извлечение текста из PDF/DOCX-фикстур и разбиение текста на чанки, включая overlap. |
| Поиск | Контракты пагинации `page/page_size` и `limit/offset`, пустой поисковый запрос. |
| Elasticsearch-клиент | Подготовка bulk-действий, поисковый запрос с фильтром по файлу, список документов через агрегацию. |

## Что ещё нужно проверить вручную

| Проверка | Ожидаемый результат |
| --- | --- |
| Загрузка реального PDF/DOCX через API | Файл парсится, разбивается на чанки и индексируется. |
| Поиск по загруженному документу | `GET /api/v1/search` возвращает найденные фрагменты с `chunk_id`, `file_name`, `page`, `text`, `score`. |
| Список документов | `GET /api/v1/documents` возвращает файлы, которые есть в индексе. |
| Полный запуск через Docker Compose | Поднимаются backend, frontend, Elasticsearch, Redis, PostgreSQL, Prometheus и Grafana. |

## Замечания

- В Elasticsearch используется analyzer `russian`; если требование `analysis-ru` будут проверять строго, это лучше уточнить отдельно.
- Redis есть в окружении, но кеширование поисковых запросов в коде пока не видно.
- Grafana provisioning добавлен, но полноценный dashboard проекта ещё нужно подготовить отдельно.
- При локальной проверке backend был доступен на `8001`, потому что порт `8000` уже был занят на машине.
- Grafana не стартовала локально из-за занятого порта `3000`; остальные сервисы, нужные для проверки загрузки и поиска, были подняты.
